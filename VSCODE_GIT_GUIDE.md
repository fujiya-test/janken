# VS CodeでのGit操作ガイド

Visual Studio Code (VS Code) を使って、コマンドを打たずにマウス操作で Git/GitHub を管理する方法やで。

## 1. 準備：プロジェクトを開く
1. VS Codeを起動する。
2. **[ファイル]** -> **[フォルダーを開く]** を選択。
3. 今回の `じゃんけん` フォルダを選択して開く。

## 2. 「ソース管理」パネルを開く
左端のメニューバーにある、**枝分かれしたアイコン（ソース管理）** をクリックするんや（ショートカットは `Ctrl + Shift + G`）。
ここが Git 操作のメイン画面になるで。

---

## 3. VS Codeでの基本操作 (Add -> Commit -> Push)

コードを書き換えて保存すると、ソース管理パネルに **「変更」** という項目が出てくるで。

### ① `git add` (変更をステージに上げる)
*   変更されたファイル名の横にある **「＋」 (変更をステージ)** アイコンをクリックする。
*   ファイルが **「ステージ済みの変更」** に移動すれば、買い物かごに入れた状態や！

### ② `git commit` (変更を記録する)
*   上の方にある **「メッセージ」** 欄に、何をしたか（例：「メッセージの修正」など）を入力する。
*   その上にある **[コミット]** ボタンをクリックする。
*   これで自分のPCへの保存は完了や！

### ③ `git push` (GitHubに送信する)
*   コミットが終わると、ボタンが **[変更を同期]** に変わるはずや。
*   そのボタンをクリックするか、左下の「くるくる回る矢印（同期）」アイコンをクリックする。
*   これで GitHub に反映されるで！

---

## 4. VS Codeの便利な機能

### 🔍 差分 (Diff) の確認
*   ソース管理パネルでファイル名をクリックするだけで、**「修正前」と「修正後」が左右に並んで表示される** で。
*   どこを直したか一目瞭然やから、ミスを防げるんや！

### 🌳 ブランチの切り替え
*   左下のステータスバーにある `main` という文字をクリックすると、別のブランチを作ったり（Create new branch）、切り替えたりできるで。

### ⌨️ ターミナルの併用
*   `Ctrl + @` (または `Ctrl + J`) を押すと、VS Code の中でターミナルが開く。
*   マウス操作でできない細かいことは、ここで今まで通りコマンドを打てばOKや！
